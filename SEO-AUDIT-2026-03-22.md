The site has strong engineering foundations and a well-crafted `llms.txt`, but is held back by a single architectural decision — `"use client"` on every public page — and a cluster of missing basics (no canonical, no H1, no schema, no privacy policy). These are all fixable, and most are low-effort.

---

## Critical Issues (fix immediately)

### 1. All public pages are Client-Side Rendered

**Files:** `app/page.tsx:1`, `app/pricing-plan/page.tsx:1`

Every indexable page starts with `"use client"`. This means:

- Google's **first-pass crawl sees a blank HTML shell** — content is indexed in a delayed second pass (days/weeks later)
- **All AI crawlers** (GPTBot, ClaudeBot, PerplexityBot) **get zero content** — they don't execute JavaScript
- The `priority` prop on the hero image has no effect because it's client-rendered

**Fix:** Refactor `app/page.tsx` into a Server Component. Extract only interactive pieces into client islands: `Navbar`, `RotatingText`, `DifferencesSection`, `ScrollStack`. Static sections need no client state.

### 2. No H1 tag on any page

**File:** `app/page.tsx` (hero section)

The hero uses `<p>` tags for all headline text. The first semantic heading is an `<h2>` in a lower section. Google uses `<h1>` as a primary relevance signal.

**Fix:** Change the hero `<p>` to `<h1>`. Tailwind classes control appearance, not the element type — zero visual impact.

### 3. No canonical tags

**File:** `app/layout.tsx`

No page sets `alternates.canonical`. The site is served at `www.tryclean.ai` but `sitemap.ts`, `robots.ts`, and `llms.txt` all reference non-www `tryclean.ai`. Both versions may be indexed separately, splitting link equity.

**Fix:** Add `alternates: { canonical: "https://www.tryclean.ai" }` to layout metadata. Fix `BASE_URL` in `sitemap.ts` and `robots.ts` to match.

### 4. No viewport meta tag

**File:** `app/layout.tsx`

The App Router viewport export is missing. Mobile browsers render at desktop width and scale down — **fails Google's mobile-friendliness test**.

**Fix:** `export const viewport: Viewport = { width: "device-width", initialScale: 1 };`

### 5. No HTTP security headers

**File:** `next.config.ts` (currently empty)

No HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, or Permissions-Policy.

### 6. No privacy policy or terms of service

The site collects emails (waitlist) and GitHub repo data. Operating without a privacy policy is a legal exposure and QRG trust failure.

---

## High Priority (fix within 1 week)

### 7. Zero structured data (schema.org)

No JSON-LD exists anywhere. The site is ineligible for all rich results. Need Organization, WebSite, and SoftwareApplication schemas at minimum.

### 8. No per-page metadata

All pages share one generic title and description. Each public page needs unique metadata + Open Graph tags.

### 9. TTFB at 1,385ms (target: <200ms)

`next.config.ts` is empty — no edge caching. Fix via `headers()` and CDN configuration.

### 10. Technical expertise buried in llms.txt

The best E-E-A-T content (tree-sitter, LanceDB, hybrid search scoring) lives in `llms.txt` which Google doesn't index. Create a `/how-it-works` page.

### 11. No About / Team page

No founders, team, or company identity visible. Major trust barrier.

### 12. Duplicate animation libraries

Both `framer-motion` and `motion` in `package.json` — same library, different names. ~100-150KB wasted.

### 13. Remotion in production dependencies

`@remotion/cli`, `@remotion/player`, `remotion` in `dependencies`, not `devDependencies`. ~300KB+ gzipped.

### 14. Incomplete sentence in Steps section

Step 02: "They all tap into the same pre-built" — ends mid-thought.

---

## Medium Priority (fix within 1 month)

### 15. sitemap.ts uses `new Date()` for lastModified

Every deploy falsely signals all pages changed. Replace with static ISO dates. Remove `changeFrequency` and `priority` (ignored by Google).

### 16. Hero content invisible without JavaScript

Hero wrapper has `opacity:0; transform:translateY(30px)`. Content requires JS to animate in.

### 17. Above-fold images incorrectly lazy-loaded

- `glass-logo.png` in hero has `loading="lazy"`
- Nav brand icon has `loading="lazy"`
- `glass-logo.png` `sizes="100vw"` should be `"(max-width: 640px) 500px, 962px"`

### 18. Zodiak font uses `display: swap`

Hero display font swaps cause FOUT and CLS. Change to `display: 'optional'`.

### 19. Unsourced/inconsistent performance statistics

"50% less token spend" vs "up to 70% token reduction" across pages and llms.txt.

### 20. Agent compatibility inconsistency

Homepage shows 4 agents, llms.txt lists 8.

### 21. Documentation not linked from navigation

`docs.tryclean.ai` exists but unreachable from any public page.

### 22. Pricing page needs explanatory copy

~200 words, below the 300-word floor.
